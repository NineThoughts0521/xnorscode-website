## 1. Hero（首页首屏）

### 模块定位说明
用于在 **3 秒内建立品牌势能**：
- 让访客瞬间理解 XnorsCode 的技术高度；
- 建立“AI + 系统级开发 + 创新实验室”的第一认知；
- 通过视觉冲击与动态语言引导继续下滑。

### 视觉布局结构
- **全屏沉浸式布局**（100vh）
- 左侧：超大标题 + 简述 + 两个 CTA（查看项目 / 联系合作）
- 右侧：3D 风格“XOR 芯片核”或数据立方体（可循环微旋转）
- 背景：低透明度代码雨 + 数据流网格 + 粒子噪声层

### 文案内容（简洁高级）
- 主标题：**XOR THE FUTURE**
- 副标题：**我们以系统级工程与 AI 能力，构建下一代数字产品。**
- 辅助语：**Code with precision. Build beyond boundaries.**
- CTA：
  - **探索项目**
  - **开启合作**

### 动效设计建议
- 标题逐字出现（字符解码效果，从 `0101` 过渡到正常文字）
- 背景数据流缓慢纵向移动（8–12s 循环，避免眩晕）
- CTA 按钮 hover：外发光扩散 + 描边亮度提升
- 鼠标轻微视差：右侧视觉核心跟随 3–5px

### UI 风格说明
- 玻璃拟态信息卡（`backdrop-blur + 低透明边框`）
- 高对比超大字重（中英混排，强调工程感）
- 发光描边与细网格线结合，呈现“控制台 + 实验室”气质

### 颜色搭配建议
- 主背景：`#06080F` / `#0A0F1E`
- 霓虹蓝：`#24D5FF`
- 科技紫：`#8B5CFF`
- 强调高光：`#C8F1FF`
- 文本灰白：`#D7DEEA`

## 2. 我们的服务（Services）

### 模块定位说明
清晰传达商业价值与技术能力，将抽象“技术强”转化为可购买、可理解、可落地的服务体系。

### 视觉布局结构
- 三列或四列卡片矩阵（桌面端），移动端改为纵向滑动卡组
- 每张卡片包含：服务名 / 一句话价值 / 技术标签
- 区块顶部加入“能力坐标轴”（速度、稳定性、可扩展性）微图形

### 文案内容（简洁高级）
建议四大服务：
1. **AI 原生应用开发**  
   从模型接入到产品化闭环，打造真正可运营的 AI 产品。
2. **系统级平台工程**  
   高并发、高可用、可观测，构建长期演进的技术底座。
3. **算法与自动化系统**  
   以算法驱动效率升级，让业务流程持续自优化。
4. **创意技术体验设计**  
   在性能与美学之间，打造具有记忆点的交互体验。

### 动效设计建议
- 卡片进入：向上 20px + 模糊清晰化（stagger 80ms）
- Hover：卡片边缘电流扫光（线性渐变从左到右）
- 图标微动：呼吸光效（opacity 0.75 ↔ 1）

### UI 风格说明
- 卡片采用玻璃拟态 + 内发光（inner glow）
- 技术标签使用胶囊 chip（细描边 + 半透明填充）
- 分割线尽量使用 1px 低透明亮线，减少厚重感

### 颜色搭配建议
- 卡片底：`rgba(18, 24, 42, 0.55)`
- 卡片描边：`rgba(120, 168, 255, 0.35)`
- Hover 光带：`linear-gradient(90deg, #24D5FF, #8B5CFF)`
- 标签文字：`#BFD8FF`

### 模块定位说明
将品牌势能转化为行动，降低咨询门槛，并传达“高端但不疏离”的合作体验。

### 视觉布局结构
- 左侧：合作引导文案 + 联系方式 + 响应时效承诺
- 右侧：极简表单（姓名 / 邮箱 / 需求类型 / 预算区间 / 项目描述）
- 底部：社交入口（GitHub / X / LinkedIn / 邮件）

### 文案内容（简洁高级）
- 标题：**Let’s Build the Next XOR.**
- 说明：**告诉我们你的野心，我们会给出系统级的实现路径。**
- 承诺：**48 小时内回复，7 天内提供初步技术方案。**
- 按钮：**发送需求**

### 动效设计建议

- 提交成功：按钮切换为“信号发送完成”动效（短暂脉冲）

- 表单保持极简、强可读，避免过度装饰

## 6. 全站统一设计系统建议

### 字体系统
- 中文：**HarmonyOS Sans SC / 思源黑体**
- 英文与数字：**Sora / Inter / JetBrains Mono（代码字段）**
- Hero 标题建议字重 700–800，正文 400–500

### 栅格与间距
- 桌面端 12 栏栅格，最大内容宽度 1280px
- 区块垂直间距 120–160px，移动端缩减为 72–96px
- 卡片圆角建议 16–20px，保持未来感与可读性平衡

### 动效原则（Motion Tokens）
- 快速反馈：120–180ms
- 常规过渡：240–320ms
- 沉浸背景循环：8–16s
- 缓动函数：`cubic-bezier(0.22, 1, 0.36, 1)`

### 工程实现建议
- 前端：Next.js + Tailwind + Framer Motion
- 3D/图形：Three.js（轻量使用，仅首屏焦点）
- 背景特效：Canvas 或 WebGL（控制透明度与帧率）
- 性能目标：LCP < 2.5s，首屏动画在低性能设备自动降级
